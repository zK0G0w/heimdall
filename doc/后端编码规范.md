# 后端编码规范

> 适用于所有新增代码。存量代码在修改时逐步对齐。

---

## 1. 包结构规范

### 1.1 业务模块标准目录

```
com.ahead.payment.{模块名}/
├── config/              配置类（@Configuration、@ConfigurationProperties）
├── constant/            常量类
├── controller/          控制器
├── model/               所有数据模型（统一入口）
│   ├── entity/          数据库实体（对应表结构）
│   ├── dto/             数据传输对象（跨层传递、查询条件、Excel 导入等）
│   ├── vo/              视图对象（返回前端的展示数据）
│   ├── req/             请求对象（Controller 方法入参）
│   └── resp/            响应对象（接口返回的复合结构）
├── mapper/              MyBatis Mapper 接口
├── service/             Service 接口
│   └── impl/            Service 实现类
└── strategy/            策略模式（如有）
```

### 1.2 model 子包使用原则

| 子包 | 用途 | 命名后缀 | 示例 |
|------|------|----------|------|
| entity/ | 映射数据库表，包含 `@TableName` | `Entity` | `PayOrderEntity` |
| dto/ | 跨层数据传递、查询条件封装、Excel 导入模型 | `Dto` | `PayOrderDto` |
| vo/ | 返回前端的展示对象、Excel 导出模型 | `Vo` | `PayOrderVo` |
| req/ | Controller 方法的请求入参（`@RequestBody` / `@ParameterObject`） | `Req` | `PayOrderAddReq` |
| resp/ | 接口返回的复合结构（非简单 VO 场景） | `Resp` | `BocAddOrderResp` |

**判断规则**：
- `@RequestBody` 或 `@ParameterObject` 入参 → `req/`
- 接口返回给调用方的结构 → `resp/`（简单场景用 `vo/` 亦可）
- 内部层间传递、与前端无直接关系 → `dto/`
- 直接映射数据库表 → `entity/`
- 纯展示/导出用途 → `vo/`

## 2. 命名规范

### 2.1 类命名

| 类型 | 后缀 | 大小写 | 示例 |
|------|------|--------|------|
| 数据库实体 | `Entity` | 首字母大写 | `PayOrderEntity` |
| DTO | `Dto` | 首字母大写 | `PayOrderDto` |
| VO | `Vo` | 首字母大写 | `PayOrderVo` |
| 请求对象 | `Req` | 首字母大写 | `PayOrderAddReq` |
| 响应对象 | `Resp` | 首字母大写 | `BocAddOrderResp` |
| Service 接口 | `Service` | - | `PayOrderService` |
| Service 实现 | `ServiceImpl` | - | `PayOrderServiceImpl` |
| Controller | `Controller` | - | `PayOrderController` |
| Mapper 接口 | `Mapper` | - | `PayOrderMapper` |

**禁止**：
- `*Server` 后缀用于 Service 实现（应为 `*Service` 或 `*ServiceImpl`）
- DTO/VO 后缀使用全大写（`UserDTO` → `UserDto`，`RoleVO` → `RoleVo`）
- 请求对象使用 `*Body`、`*Param` 等非标准后缀

### 2.2 包名

- 全小写，单词间不分隔
- 模块名与 Maven artifactId 中 `payment-` 后的部分一致

## 3. 类注释规范

所有新建 Java 文件必须添加类级注释：

```java
/**
 * @Description: 类的实际用途描述
 * @Author: {开发者姓名}
 * @Date: yyyy/MM/dd
 */
```

- 放置于 `import` 语句之后、类定义之前
- `@Description` 必须填写，禁止留空
- `@Author` 填写实际开发者姓名
- 存量代码在修改时补充，不做全量补加

---

## 4. 依赖注入规范

统一使用构造器注入：

```java
@RestController
@RequiredArgsConstructor
public class PayOrderController {
    private final PayOrderService payOrderService;
    private final PayChannelService payChannelService;
}
```

**禁止**：
- `@Autowired` 字段注入
- `@Resource` 字段注入
- `@AllArgsConstructor`（使用 `@RequiredArgsConstructor` 替代，只注入 `final` 字段）
- 同一类中混用多种注入方式

---

## 5. Controller 编码规范

### 5.1 类级注解（必须）

```java
@RestController
@RequiredArgsConstructor
@RequestMapping("/payOrder")
@Tag(name = "支付订单管理")
@SecurityRequirement(name = HttpHeaders.AUTHORIZATION)
public class PayOrderController {
```

### 5.2 方法级注解

```java
@SysLog("新增订单")
@SaCheckPermission("business_payOrder_add")
@PostMapping
public R<Boolean> addOrder(@RequestBody PayOrderAddReq req) {
```

- 权限控制：`@SaCheckPermission("模块_资源_操作")`
- 操作日志：`@SysLog("操作描述")`
- 字典翻译：`@ResultTranslatebData`（需要时添加）

### 5.3 返回值

- 所有接口统一返回 `R<T>`，禁止返回裸类型
- 文件下载等特殊场景可返回 `void`，需在方法注释中说明

## 6. Service 编码规范

### 6.1 接口定义

```java
public interface PayOrderService extends IService<PayOrderEntity> {
    R<Boolean> addOrder(PayOrderAddReq req);
}
```

- `@Service` 注解只加在实现类上，接口不加

### 6.2 实现类

```java
@Slf4j
@Service
@RequiredArgsConstructor
public class PayOrderServiceImpl extends ServiceImpl<PayOrderMapper, PayOrderEntity>
        implements PayOrderService {
    private final PayChannelService payChannelService;
}
```

## 7. Import 规范

- 禁止使用全限定类名内联（如 `implements java.io.Serializable`），必须通过 `import` 导入
- 禁止保留未使用的 import
- 禁止使用通配符 import（`import xxx.*`），逐个导入

---

## 8. 注释规范

- 业务方法中的关键逻辑添加简洁注释，说明业务意图和边界处理
- 禁止无意义的冗余注释（如 `// 获取用户` 紧跟 `getUser()`）
- 禁止保留注释掉的代码块，废弃代码直接删除

---

## 9. 数据库实体规范

```java
@Data
@TableName("pay_order")
@Schema(description = "支付订单")
public class PayOrderEntity {

    @TableId(type = IdType.ASSIGN_ID)
    @Schema(description = "主键ID")
    private Long id;

    @Schema(description = "订单编号")
    private String orderNo;
}
```

- 必须添加 `@TableName`
- 必须添加 `@Schema(description = "...")` 用于 API 文档
- 主键使用 `@TableId(type = IdType.ASSIGN_ID)`
- 字段使用 `@Schema` 注解说明含义

---

## 10. 异常处理规范

- 业务异常统一抛出 `CustomException`，由 `GlobalExceptionAdvice` 统一处理
- 禁止在 Controller 层 try-catch 后返回自定义错误结构
- 订单相关异常使用 `OrderException`
- 权限异常使用 `PermissionException`
- 登录异常使用 `LoginException`
